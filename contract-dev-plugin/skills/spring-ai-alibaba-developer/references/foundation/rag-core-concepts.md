# RAG 核心概念

## 概述
RAG（Retrieval-Augmented Generation，检索增强生成）是 SAA 的核心能力模块，通过检索外部知识增强 LLM 生成能力。

## 核心架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      RAG 完整架构                               │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────┐   ┌─────────────┐   ┌─────────┐   ┌────────────┐  │
│  │ Reader  │ → │ Transformer │ → │ Vector  │ → │ Retrieval  │  │
│  │ (读取)  │   │ (转换分割)   │   │ Store   │   │ (检索)     │  │
│  └─────────┘   └─────────────┘   └─────────┘   └────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

## 检索时机

| 模式 | 触发方式 | 适用场景 |
|------|---------|---------|
| **Advisor 自动注入** | 每次 ChatClient 调用自动触发 | 企业知识库、文档助手、固定知识域 |
| **Agent 智能决策** | Agent 分析问题后决定是否调用 Tool | 智能客服、代码助手、多类型问答 |

## 应用场景

| 场景 | 推荐模式 | 说明 |
|------|---------|------|
| 企业知识库问答 | Advisor 自动注入 | 用户提问即触发检索 |
| 产品手册/文档助手 | Advisor | 固定知识域，每次都需要检索 |
| 智能客服 | Agent + Tool | 根据问题类型决定是否检索 |
| 代码助手 | Agent | 代码问题检索文档，其他问题直接回答 |
| 法律/医疗咨询 | Advisor + Rerank | 高精度要求，必须检索+重排序 |
