# OpenSpec 文档与代码一致性审查

## 概述

每次 `/reviewer` 都会通过 agent team 执行此审查。
agent-1 读代码、agent-2 读文档，主线程对比输出报告。

---

## Agent 检查项参考

### Agent-1（代码阅读者）需整理的内容

| 维度 | 说明 |
|------|------|
| 工具/方法参数 | 名称、类型、是否必填 |
| 返回数据结构 | 顶层字段、嵌套结构、数组层级 |
| 核心业务规则 | 判断条件、枚举值使用、分支逻辑 |
| 触发路径 | LLM 如何调用到该工具，提示词中的触发关键词 |

### Agent-2（文档审查者）需检查的内容

**一致性检查：**

| 检查点 | 说明 |
|--------|------|
| 接口参数 | 文档中的参数名/类型与代码定义是否一致 |
| 返回结构 | 文档描述的响应字段与实际返回 JSON 是否匹配 |
| 业务规则 | 文档中的判断条件、枚举值与代码逻辑是否一致 |
| 工具/方法名 | 文档中引用的方法名是否与代码一致（重命名未同步） |
| 触发条件 | 文档中描述的触发场景与代码实际调用路径是否匹配 |

**过时内容检查：**

| 检查点 | 说明 |
|--------|------|
| 已删除的功能 | 代码已删除但文档仍描述该功能 |
| 已重构的实现 | 实现方式变了但文档仍是旧方案 |
| 已更新的枚举 | 新增/删除枚举值但文档未同步 |

**规范性检查：**

| 检查点 | 说明 |
|--------|------|
| 必要章节完整 | 是否有概述、设计说明、任务列表 |
| 描述清晰 | 是否存在"待定"、"TODO"、"暂时先这样"等模糊描述 |
| 任务完成状态 | `tasks.md` 中是否有未勾选的 `- [ ]` 任务 |
| 增量规格同步 | `openspec/changes/<name>/specs/` 是否已同步到 `openspec/specs/` |

---

## 审查报告格式

```markdown
## OpenSpec 文档审查报告

### 变更名称：<change-name>

#### ✅ 一致项
- xxx

#### ⚠️ 不一致项（需修复）
- **[参数名不一致]** 文档写 `queryScope`，代码参数名为 `scope`
- **[返回结构缺失]** 文档未描述 `records[0].contracts` 嵌套结构

#### 📋 过时内容（需更新）
- **[已删除功能]** 文档第3节描述了 `SCOPE_ALIAS` 映射，代码已删除

#### 📝 规范问题
- **[描述模糊]** tasks.md 第2项描述为"待定"，需明确
- **[未完成任务]** tasks.md 中有 2 个 `- [ ]` 未完成任务
```

---

## 注意事项

- 两个 agent 只做信息收集，**不修改任何文件**
- 无活跃 openspec 变更时，也要检查 `openspec/specs/` 主规格是否与代码一致
- 发现不一致时，列出问题清单后**询问用户**：是修复代码还是修复文档？不要自动修复
