# 集成测试编写规范

## 概述

适用于任何包含集成测试的项目。核心原则：测试要有层次，验证要宽松，职责要清晰，避免冗余。

---

## 一、测试文件职责划分原则

每个项目应明确"核心集成测试文件"，所有场景集中在此验证，不要散落在多个文件中。

**应删除的冗余测试类型**（审查时应指出）：

| 冗余类型 | 原因 |
|---------|------|
| 只验证 Bean 注入/应用启动的测试 | 隐含在任何集成测试运行中 |
| 只验证工具/方法被调用、不验证参数和输出的测试 | 粒度太粗，无法发现数据错误 |
| 与核心集成测试完全重叠的独立测试文件 | 维护两份相同覆盖，成本加倍 |
| 对同一场景建立多个独立测试类 | 应合并，保持单一入口 |

**单元测试保留原则**：独立于外部环境、验证纯逻辑规则（如参数映射、路径查找、枚举转换）的单元测试具有不可替代的价值，不应被集成测试合并。

---

## 二、两层验证规范

集成测试的每个方法应同时覆盖两层，缺少任何一层均为不完整：

```
第一层：行为验证 —— 系统是否做了正确的事？
        （调用了哪个工具/方法，传了什么参数）

第二层：输出验证 —— 系统是否返回了正确的数据结构？
        （响应字段是否存在，结构是否符合预期）
```

**示例（伪代码）：**

```java
@Test
void queryContract_shouldReturnCorrectData() {
    // 执行
    String result = ask("查询合同C123");

    // 第一层：行为验证
    assertToolCalled("ontologyQuery");
    assertToolParam("ontologyQuery", "entity", "Contract");

    // 第二层：输出验证
    assertOutputField("queryEntity", "Contract");
    assertOutputHasRecords();
    assertFirstRecordHasField("contractCode");
}
```

**审查要点**：若测试只有行为验证没有输出验证，要求补充第二层。

---

## 三、宽松验证策略（避免维护成本剧增）

| 验证对象 | 推荐策略 | 原因 |
|---------|---------|------|
| 系统元数据（如 `queryEntity`） | **精确匹配** | 框架级字段，极稳定 |
| 业务响应字段 | **只验证存在，不验证值** | 业务数据随环境/时间变化 |
| ID 回显字段（输入值原样返回） | **精确匹配** | 值由查询输入决定，不随数据变化 |
| 深层嵌套结构 | **只验证第一层展开字段存在** | 防止深层变化导致脆断 |
| 固定枚举配置值 | **精确匹配** | 配置稳定，值变化即为 bug |

**审查要点**：若测试对业务数据做精确断言（如 `contractAmount == 120000`），应改为只验证字段存在。精确匹配仅用于稳定不变的值。

---

## 四、新增功能时的测试模板

```java
@Test
void feature_shouldDescribeExpectedBehavior() {
    // 执行
    String result = invoke("触发该功能的输入");

    // 第一层：行为验证
    assertCorrectMethodCalled();
    assertCorrectParamsPassed();

    // 第二层：输出验证
    assertOutputStructureValid();       // 元数据/结构正确
    assertOutputHasData();              // 有返回数据
    assertKeyFieldExists("关键字段");   // 关键字段存在（不验证值）
}
```

---

## 五、常见审查问题

| 问题 | 现象 | 修复方向 |
|-----|------|---------|
| 只有行为验证 | 测试通过但输出数据错误 | 补充第二层输出验证 |
| 精确断言业务数据值 | 数据更新后测试频繁失败 | 改为只验证字段存在 |
| 测试文件职责重叠 | 同一场景在多个文件中测试 | 合并到核心集成测试文件 |
| 独立启动验证测试 | `applicationContext_shouldLoad` 等 | 直接删除 |
| 单元测试被集成测试替代 | 纯逻辑规则无专项单元测试 | 保留/补充独立单元测试 |
